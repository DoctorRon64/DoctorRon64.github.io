App development


This was a project from my first year in game development. I learned how to work with the .NET MAUI framework and gained insight into app development. What I also noticed was that games aren't ideally suited for being apps because apps aren't designed for continuous update loops, but rather for executing actions based on user interactions through events.

For this assignment, we had to make one of those Tamagotchi games. The requirements were that the pet had to make a Tamagotchi app with the .NET MAUI framework, where we could take care of our pet daily. The pet can get hungry, then it needs food; thirsty, then it needs a drink; bored, then it needs attention; lonely, then it needs affection; overstimulated and tired. For the assignment, I wanted to make a fun game that the user would enjoy. On the main screen, you can input the name of your pet, which will then be stored in the database and synced with it. You can also press the delete button to remove your pet.

For the layout, I designed each page to display the values the pet needs, with a status text at the top and the current pet needs listed at the bottom. You can navigate between each page from any other page. I ensured that the app is scalable for different resolutions and assigned a distinctive color to each value of the pet's needs. Food is red / brown, drinks are blue, tired / sleep is dark green dark because of night time, pink is for love I switched that from the stimulated value because i found that funny, orange is energetic fits well with gaming and for arcade, and purple is loneliness I choose this because its an mysterious emotion and color.
