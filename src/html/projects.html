Game Architecture

Date: 22/09/2023


Contributors:
Group of 2

Context:
In this school project we had to make a game that had only one MonoBehaviour (unity's base class). It also had to contain at least 2 design patterns. Me and one of my class mates made a game containing an object pool for shooting, observer patterns for detection and the factory pattern for instantiating.

Learning:
I learned a lot about programming patterns, how to make a uml diagram, how to structure code and how to make code efficiently. This module was one of the best modules I had and I gained a lot of insight in how to become a better developer.

Building Playful worlds: Topdown shooter

Date: 05/10/2023


Contributors:
Solo

Context:
The project involves creating a top-down or twin-stick shooter game. It must include various gameplay mechanics such as player movement and aiming with the mouse, enemy AI using Finite State Machine, ability system (passive or active), pickups, combat mechanics, and clear win/lose conditions. Additionally, the game should have a visually appealing design, a start menu, and may incorporate extra features like remappable buttons, quick select UI, different game modes, or a saving/loading system.
